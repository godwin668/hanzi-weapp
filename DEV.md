# 开发指南

## 环境要求

- Node.js >= 16
- 微信开发者工具（用于小程序预览和调试）

## 安装依赖

```bash
npm install
```

## 编译命令

| 命令 | 说明 |
|------|------|
| `npm run build:weapp` | 一次性编译微信小程序 |
| `npm run build:h5` | 一次性编译 H5 网页版 |
| `npm run dev:weapp` | 微信小程序监听模式（改代码自动编译） |
| `npm run dev:h5` | H5 监听模式（改代码自动编译） |

## 微信小程序开发流程

### 1. 启动自动编译

```bash
npm run dev:weapp
```

该命令会持续监听 `src/` 目录下的文件变化，自动重新编译到 `dist/` 目录。

### 2. 导入微信开发者工具

打开**微信开发者工具** → 导入项目 → 选择 `D:\projectsai\hanzi` 根目录。

`project.config.json` 中已配置 `miniprogramRoot: "dist/"`，开发者工具会自动识别编译输出目录。

### 3. 开发调试

- 修改 `src/` 下的源码，保存后自动触发编译
- 微信开发者工具会自动刷新预览

## H5 网页版开发流程

```bash
npm run dev:h5
```

编译后在浏览器打开 `dist/index.html` 即可预览。

## 常见问题

### Q: 每次改代码都要重新编译吗？

A: 如果用的是 `build:weapp`（一次性编译），需要手动重新运行。建议使用 `dev:weapp`（监听模式），改代码后自动编译。

### Q: 微信开发者工具导入哪个目录？

A: 导入项目根目录 `hanzi/`，工具会自动识别 `dist/` 为小程序根目录。**不要直接导入 `dist/` 目录**。

### Q: 云开发功能不可用？

A: 需要在微信小程序后台开通云开发，并绑定环境 ID。在开通之前，云函数调用会自动降级为本地 mock 数据。
